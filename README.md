# 🌟 Планетарий «Урания»

Информационная система для управления программами показов и журналом астрономических наблюдений планетария. Приложение работает одновременно с двумя базами данных: **PostgreSQL** (основные данные) и **MongoDB** (расширенные метаданные наблюдений).

## Возможности

- **Управление программами показов** — добавление, редактирование, мягкое удаление, просмотр статистики по сеансам и выручке
- **Журнал наблюдений** — ведение записей с привязкой к астрономическому объекту, телескопу и ответственному астроному
- **Метаданные наблюдений (MongoDB)** — гибкое хранение детальных погодных данных, параметров астрофотографии, снимков, измерений и списка участников
- **Визуальная индикация** — строки без метаданных выделяются красным, с метаданными — зеленым
- **Ролевая система доступа** — три роли с разграничением прав на уровне приложения и СУБД

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/Dashh-B/Planetarium_Urania.git
cd Planetarium_Urania
```

### 2. Настройка PostgreSQL

Убедитесь, что PostgreSQL запущен на порту `4242`. Выполните дамп для создания схемы и начального наполнения базы данных:

```bash
psql -U postgres -p 4242 -f DUMP/postrge_dump.sql
```

Дамп создает:
- базу данных `postgres` с десятью таблицами
- четыре учетные записи: `urania_auth`, `urania_viewer`, `urania_manager`, `urania_admin`
- тестовых пользователей приложения

> **Пароли учетных записей PostgreSQL** задаются в файле `Core/DatabaseHelper.cs` в словаре `ConnectionStrings`. При необходимости замените значения паролей на свои и пересоздайте учетные записи с теми же паролями в PostgreSQL.

### 3. Настройка MongoDB

Убедитесь, что MongoDB запущен на порту `27017`. Для восстановления данных из дампа используйте `mongorestore`:

```bash
mongorestore --port 27017 DUMP/mongodump/
```

Либо импортируйте коллекцию вручную:

```bash
mongoimport --port 27017 --db planetarium_urania \
  --collection observation_metadata \
  --file DUMP/planetarium_urania.observation_metadata.json \
  --jsonArray
```

### 4. Сборка и запуск

Откройте решение в Visual Studio 2026 и запустите проект (`F5`), или выполните сборку через CLI:

```bash
dotnet run
```

## Архитектура хранения данных

Система использует подход **полиглотного хранения данных**:

**PostgreSQL** хранит строго структурированные данные с жесткими связями:
`programs` → `sessions`, `observations` → `astronomical_objects`, `telescopes`, `employees`, `visitors` → `tickets`

**MongoDB** хранит гибкие метаданные наблюдений (коллекция `observation_metadata`), связанные с PostgreSQL через поле `observation_id`. Структура документа варьируется в зависимости от типа наблюдения: от подробных параметров астрофотографии до краткой текстовой заметки.

---

## Учетные записи для тестирования

Тестовые пользователи создаются дампом PostgreSQL. Учетные данные можно найти в таблице `app_users` после применения дампа. Пароли хранятся в виде SHA-256 хэша.
